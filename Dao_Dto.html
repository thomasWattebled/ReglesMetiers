Les appels des Dao.

Les Dao ne sont pas appelés directement dans d'autre classes. En effet chaque Dao implémente sa propre interface (exemple DaoClient implémente IDaoClient). Ce sont les interfaces qui sont ensuite appelés dans les autres classes.
Les Dao sont des attributs dans les services.
Les dao sont utilisés dans les classes Service... mais non les DTO.

Explication d'une utilisation d'un Dao.

Un Dao implémente toujours une interface (DaoClient implémente IDaoClient , DaoTache implémente IDaoTache).
Le Dao a aucune référence en dehors de sa classe, c'est l'interface qui est utilisé.

private IDaoClient daoClient  ```.
On retrouve c'est différent appels dans des classe qui ont pour nom Service... (ServiceActivite , ServiceClient , ServicePc...).
Tous ces classes implémente également une interface (ServiceClient implémente IServiceClient, ServiceSocle Implémente IServiceSocle). Ce sont de nouveaux les interfaces qui sont utilisés dans d'autres classes.
On retrouve plusieurs appels dans des classes différentes mais toutes ces classes suivent le même schema.
